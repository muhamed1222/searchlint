# ExecPlan 457: Stabilize Release Evidence Readiness Board Link

## Purpose

`docs/RELEASE_EVIDENCE_READINESS_SUMMARY.md` links readers to
`docs/RELEASE_EVIDENCE_CONTROL_BOARD.md`, but the summary is generated by
`scripts/verify-release-evidence-readiness.mjs`. A fresh
`pnpm final-release:gate` run removes the manual link unless the generator emits
it.

This plan keeps the orchestration link reproducible without changing any release
status, counter, or evidence semantics.

## Scope

- Update `scripts/verify-release-evidence-readiness.mjs` so generated readiness
  markdown includes the control board routing note.
- Regenerate `docs/RELEASE_EVIDENCE_READINESS_SUMMARY.md`.
- Keep final release status blocked until real external evidence exists.

## Non-goals

- No owner, reviewer, legal, provider, publication, deployment, security,
  backup, or final release evidence.
- No gate closure.
- No generated report counter changes.
- No package, lockfile, CI, source package, or release action changes.
- No `v1.0.0` tag or publication action.
