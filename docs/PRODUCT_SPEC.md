# SearchLint Product Spec

Status date: 2026-06-30

This document reconciles the required repository reading set with the current
published `main` branch. It does not approve SearchLint 1.0 release and does not
reduce the product scope described in `SEARCHLINT_1_0_DEVELOPMENT_PLAN.md`.

## Product

SearchLint is a deterministic, rule-based SEO diagnostics product for local
developer feedback, CI enforcement, editor diagnostics, crawler-based site
analysis, and cloud/dashboard workflows.

SearchLint 1.0 is the full product described in
`SEARCHLINT_1_0_DEVELOPMENT_PLAN.md`, including:

- SearchLint language and route contracts.
- Shared rule engine and diagnostic engine.
- Source, raw HTML, rendered DOM, HTTP, crawler, Google, and Yandex evidence.
- Next.js development badge and overlay.
- CLI, CI, reporters, LSP, and VS Code integration.
- Cloud API, workers, dashboard, storage, auth, billing, reports, notifications,
  observability, and agency mode.

## Non-Goals Before Final Release Evidence

The repository must not treat the current local developer product, beta package
chain, generated reports, static deployment contracts, or sample evidence as a
SearchLint 1.0 release.

Until `pnpm final-release:gate` passes, SearchLint must not:

- create `v1.0.0`;
- publish final npm packages;
- publish the VS Code extension as SearchLint 1.0;
- launch or announce SearchLint 1.0;
- claim missing external, legal, reviewer, provider, publication, or live
  deployment evidence.

## Product Invariants

- Production user sites must receive 0 KB of SearchLint production code.
- Production user sites must make 0 SearchLint production HTTP requests.
- Production user sites must register 0 SearchLint production background
  processes or runtime hooks.
- Rule execution and severity calculation must not require AI.
- The same deterministic rule engine must power overlay, CLI, IDE/LSP, crawler,
  CI, and cloud surfaces.
- Every diagnostic must include a stable rule ID and evidence.
- File, line, selector, provider observation, and external freshness data may be
  shown only when proven.
- Blockers must not rely only on subjective heuristics.

## Current Release Boundary

The current repository has verified local developer and static contract
evidence, but SearchLint 1.0 remains blocked on external evidence. The
authoritative release status is generated by:

- `pnpm release:evidence-readiness`
- `pnpm final-release:gate`

The current gate summary is recorded in:

- `docs/RELEASE_EVIDENCE_READINESS_SUMMARY.md`
- `docs/SEARCHLINT_1_0_FINAL_RELEASE_GATE.md`
- `docs/RELEASE_OWNER_ACTION_RUNBOOK.md`

## Acceptance

A product change is acceptable only when it preserves the full SearchLint 1.0
scope, keeps the product invariants true, updates the relevant documentation or
contract, and passes the validation required by the active ExecPlan.
